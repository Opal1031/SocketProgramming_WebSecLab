# STEP4: 해킹 및 보안 패치

## 활동 내용
- 개발자 도구, 프록시 등으로 직접 취약점(XSS, IDOR, 세션 조작 등) 공격 실습
- 실제 공격 성공 여부 확인
- 각 취약점에 대한 보안 패치 적용
- 보안 보고서 작성(공격 시나리오, 패치 방법 등)

## 핵심 보안/학습 포인트
- XSS, IDOR, 세션 조작 등 주요 웹 취약점의 원리와 공격 방법 이해
- 취약점이 서비스 전체에 미치는 영향 체험
- 보안 패치(입력값 검증, escape, 인증 등) 적용 실습
- 보안 보고서 작성 역량 강화

## 실습 예시
- XSS: `<img src=x onerror="alert('XSS!')">` 입력 후, 여러 사용자에게 팝업 전파 확인
- IDOR: 메시지/사용자 정보 등 비인가 접근 시도 및 결과 확인
- 세션 조작: 쿠키 변조, 인증 우회 등 실습
- 각 취약점에 대해 패치 적용 후 재공격 시도

## 실행 방법

1. Python 백엔드 서버 실행
   ```
   python app.py
   ```
2. 프론트엔드(React) 개발 서버 실행
   ```
   npm run dev
   ```
3. 브라우저에서 안내된 주소로 접속 후 실습 진행
